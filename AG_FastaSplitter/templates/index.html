Instructions

The FASTA Splitter tool takes a multi-sequence FASTA dataset (either uploaded as a file or pasted into the form) and breaks it up into separate files. This is extremely useful for running batch operations on bioinformatics prediction servers that impose sequence limits per submission.

How to use the tool:

  1. Provide Sequence Data: Either copy and paste your FASTA sequences into the text area under the Submission tab, click Choose File to upload a FASTA formatted file directly, or upload a compressed .zip archive containing your sequence file (strongly recommended for datasets greater than 500 MB).
  2. Specify Split Limit: Enter the maximum number of sequences you want in each individual split file. For instance, if you have 100 sequences and set the limit to 20, the tool will divide them into 5 files containing exactly 20 sequences each.
  3. Submit and Download: Click the Submit button. The server will validate the format, process the sequences, package the output files into a compressed .zip file, and initiate an automatic download in your browser.
  4. Reset: Click Clear fields to reset the form and start a new job.

Valid FASTA Format Rule:

Your data must strictly follow the standard FASTA rules. Each sequence must begin with a single-line header description starting with a > character, followed immediately by lines of sequence data:

>sequence_id_1 description of first sequence MDAMKRGLCCVLLLCGAVFVSPSQEIHARFRRGARSYQVICRDEKTQMIYQQHQSWLRPV LRSNRVEYCWCNSGRAQCHSVPVKSCTEPRCFNGGTCQQALYFSDFVCQCPEGFAGKCCE >sequence_id_2 description of second sequence MNSVTVSHAPLIILGLVALVGAASVAGSVVVPGLKLDVEKLTVSLQGLAGKVSVDVNKLA

About FASTA Splitter

FASTA Splitter is a high-throughput sequence partitioning utility designed to simplify workflows in structural biology, genomics, and immunoinformatics.

Why Split FASTA Files?

Many state-of-the-art bioinformatics servers (such as NetMHC, SignalP, BepiPred, and TMHMM) limit the number of sequence entries or total amino acids allowed per single submission to manage server compute load fairly. Researchers working with large-scale sequencing runs or entire proteomes often have to manually copy-paste sequences in blocks of 50 or 100.

This tool automates that tedious workflow. By splitting your massive FASTA database into exact chunk sizes, you can rapidly submit them across parallel tabs or use batch API endpoints without violating third-party resource constraints.
